Histogram

Histogram (clip, string ''mode'')

Ce filtre ajoute un histogramme de la luminance à la droite du clip considéré.

A partir de v2.5, ce filtre montre aussi les couleurs valides et non valides pour les formats YV12/YUY2. Les valeurs non valides (en dessous de 16 et au dessus de 235) sont colorées avec un ton marron/jaunâtre.

A partir de v2.53:
Un paramètre optionnel mode a été ajouté pour afficher des informations supplémentaires sur le clip. Mode peut être "classic" (classique) (le mode par défaut précédent), "levels" (niveaux), "color" (couleur)
et depuis v2.54 "luma" (luminance), "stereo", ou "stereooverlay".

Mode Classic (classique)

Cela rajoute à la droite de la video un graphe pour chaque ligne. Le coté gauche du graphe représente luma (luminance) = 0 et le coté droit représente luma (luminance) = 255. La plage de valeurs valides pour le standard CCIR601 est affichée dans une couleur légèrement différente, et une ligne verte représente Y = 128.

Disponible dans les modes YUY2 et YV12.

Modes Levels (niveaux)

Ce mode affiche trois graphes des niveaux à la gauche de l'image. Cela montre la distribution des composantes Y, U et V de chaque image.

Le graphe le plus haut représente la distribution de luminance (Y) de l'image, où la partie gauche représente Y = 0 et la partie droite représente Y = 255. La plage de valeurs valides pour le standard CCIR601 est affichée dans une couleur légèrement différente, et une ligne pointillée représente Y = 128.

Le graphe du milieu représente la distribution de la chrominance U , et le graphe du bas représente la distribution de la chrominance V.

Disponible seulement dans le mode YV12.

Mode Color (couleur)

Ce mode affiche la position de la couleur créée par U/V dans un graphe bidimensionnel situé à gauche de l'image.

La composante U est représentée sur l'axe horizontal (X), avec la partie gauche représentant U=0 et la partie droite U = 255.

La composante V est représentée sur l'axe vertical (Y), avec la partie haute représentant V=0 et la partie basse V = 255

Disponible seulement dans le mode YV12.

Mode Luma (luminance)

Ce mode amplifie la luminance et affiche les très petites variations de luminance. C'est pratique pour détecter les blocs ou le bruit, et cela peut être utile pour ajuster les paramètres d'un filtre. Dans ce mode, une différence de 1 pixel est représentée par un pixel de luminance égale à 16, accentuant donc beaucoup les petites variations. 

Disponible dans les modes YUY2 et YV12.

Modes Stereo et StereoOverlay

Ce mode affiche un graphe stéréo classique du flux audio du clip. Certains peuvent avoir vu ceci dans les studios d'enregistrement. Ces modes peuvent être utilisés pour observer la distribution gauche-droite et la distribution de la phase du signal d'entrée. StereoOverlay va superposer le graphe sur l'original. Chaque image contient seulement l'information de l'image en cours jusqu'au début de l'image suivante. Le signal est linéairement ré échantillonné 8x, pour fournir un résultat plus visible.

Ce mode requiert un signal d'entrée stéréo. Le clip d'entrée de StereoOverlay doit être dans le format YV12 seulement.

Changements:

v2.53 Ajout des différents modes.
v2.55 Ajout de points pour le mode = "stereo" pour mettre en évidence les facteurs d'échelles et les décalages.
v2.56 Ajout des couleurs non valides dans le mode YUY2.
