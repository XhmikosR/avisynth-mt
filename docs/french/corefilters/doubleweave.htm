DoubleWeave (clip)

Si le clip d'entrée est basé sur des champs, le filtre DoubleWeave agit comme Weave, sauf qu'il produit un nombre d'image deux fois plus important: au lieu de rassembler les champs 0 et 1 dans l'image 0, les champs 2 et 3 dans l'image 1, ..., il rassemble les champs 0 et 1 dans l'image 0, les champs 1 et 2 dans l'image 1, etc... Il ne change donc pas le taux de rafraichissement des images ni le nombre d'image.

Si le clip source est basé sur des images, le filtre agit comme si vous lui aviez appliqué SeparateFields avant.

Weave est en fait un raccourci de DoubleWeave suivi de SelectEven.

La plupart du temps, vous utiliserez SelectOdd ou Pulldown après avoir utilisé ce filtre, sauf si vous souhaitez réelement une video à 50fps ou 60fps. Cela peut parâitre inefficace d'entrelacer chaque paire de champs seulement pour garder la moitié ensuite. En fait, AviSynth calcul seulement les images demandées, les image non nécessaire ne sont en fait pas calculées.

Si vous manipulez une video basée sur des champs, tel qu'un flux video de camescope, vous n'aurez probablement pas besoin de ce filtre. Mais si vous manipulez une video NTSC qui a été convertit depuis un format film et que vous souhaitez appliquer le filtre Pulldown, vous devez utiliser DoubleWeave avant. Voir la description du filtre Pulldown pour plus d'explications.

Si vous manipulez une video PAL convertit depuis du film, vous n'avez pas besoin de Pulldown, mais vous aurez peut être besoin de DoubleWeave dans la situation suivante: 

# Reproduit la fonction du filtre VirtualDub "PAL deinterlace"
DoubleWeave
SelectOdd
