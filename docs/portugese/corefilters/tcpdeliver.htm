TCPServer / TCPSource

TCPServer (clip, int "port")
TCPSource (string hostname, int "port", string "compression")

Este filtro permite enviar clipes em sua rede. Você pode conectar vários clientes à mesma máquina.

Sintaxe

Server:

TCPServer (clip, int "port")

Isto gerará um thread de servidor na máquina atual pela porta especificada. Porta padrão é 22050.
Você terá saída na aplicação em que abre seu escrito, mas o servidor só estará rodando quando a aplicação (vdub por exemplo) está aberta.

Exemplo:

Colorbars(512, 256)
TCPServer()

inicia um servidor.

Cliente:

TCPSource (string hostname, int "port", string "compression")

Isto conectará à máquina com o determinado endereço (número do IP por exemplo) para um servidor que roda na determinada porta. Porta padrão é 22050 aqui também.

Compression permite a escolher a compressão usada para o vídeo:
Tipo de Compressão  Descrição
None Não usar compressão. Opção mais rápida – o vídeo não será comprimido antes de ser enviado à rede.
LZO Usar dicionário de compressão LZO. Razoavelmente rápido, mas só comprime bem em fontes artificiais, como desenhos e animes com superfícies muito uniformes.
Huffman Usa uma rotina Huffman bastante lenta por Marcus Geelnard. Comprime vídeo natural melhor que LZO.
GZip Usa Huffman Gzip só compressão. Trabalha como o Huffman, mas parece mais rápido.

Se nenhuma compressão for determinada, GZip é atualmente usado como padrão. Material entrelaçado comprime pior que desentrelaçado devido a deltaencoding para baixo. Se a velocidade da rede é um problema você pode usar SeparateFields.

Exemplo:

TCPSource("127.0.0.1")
Info()

Isto conectará à máquina local, se um servidor está rodando.

Exemplos

Você pode usar isto para rodar cada/alguns filtros em PCs diferentes. Por exemplo:

# Clustermember 1:
AVISource
Deinterlacer
TCPServer

# Clustermember 2:
TCPSource
Sharpener
TCPServer

# Clustermember 3:
TCPSource
# client app -> video codec -> final file
