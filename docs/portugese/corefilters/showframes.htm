ShowFrameNumber

ShowFrameNumber (clip, bool "scroll", int "offset", int "x", int "y", string "font", int "size", int "text_color", int "halo_color")

ShowFrameNumber puxa texto em todo quadro, indicando qual número o AviSynth pensa que é. Isto às vezes é útil em escritos. Se você aplicar filtros adicionais ao clipe produzido por ShowFrameNumber, eles tratarão o texto na quadro da mesma maneira que eles tratam uma imagem, assim os números podem ser destorcidos até que você os veja.

Se scroll (padrão: false) é fixado true o número do quadro só será puxado uma vez no vídeo e rolado de cima para baixo, puxado ao lado direito tanto quanto para se ajustar. Para o material do primeiro campo do topo o número do quadro será puxado ao lado esquerdo do clipe, para o material do primeiro campo inferior à direita e para o material baseado em campo será puxado alternadamente ao lado esquerdo e direito do clipe (dependendo se o campo é de topo ou de fundo).

A partir da v2.56 outras opções (offset, x, y, first_frame, last_frame, font, size, text_color, halo_color) estão presentes, veja Subtitle para uma explicação destas opções. offset  permite ao usuário incluir um compensador ao número do quadro mostrado.

ShowSMPTE

ShowSMPTE (clip, float "fps", string "offset", int "offset_f", int "x", int "y", string "font", int "size", int "text_color", int "halo_color")

ShowSMPTE é semelhante a ShowFrameNumber mas exibe o timecode SMPTE (horas:minutos:segundos:quadros). A partir da v2.53 não é requerido o argumento fps, a menos que o fps atual não possa ser usado. Caso contrário, o argumento fps é requerido e deve ser 24, 25, 30, ou 29.97. A partir da v2.56 outras opções (offset, offset_f, x, y, first_frame, last_frame, font, size, text_color, halo_color) estão presentes, veja Subtitle para uma explicação destas opções. offset, permite ao usuário incluir um compensador ao timecode, enquanto offset_f permite ao usuário incluir um compensador timecode que especifica o número de quadros (offset tem precedência sobre offset_f).

Tome cuidado: ambos os filtros são devidos a text-drawing bastante lento.

drop-frame x non-drop-frame timecode

Se o framerate do clipe está entre 29.969 e 29.971 o drop-frame timecode é habilitado. Originalmente, quando o sinal da TV era preto e branco, NTSC rodava a 60 Hz (30 fps). Quando incluíram cor, mudaram para 59.94 Hz (29.97 fps) por razões técnicas. Eles rodam 1000 quadros, mas contam 1001 (eles de fato nunca perdem um quadro, só um número de quadro). Os primeiros dois quadros são perdidos em cada minuto menos o décimo, ou seja 00:00:00:00, 00:00:00:01, 00:00:00:02,..., 00:00:59:29, 00:01:00:02, 00:01:00:03, 00:01:59:29, 00:02:00:02, 00:02:00:03, ..., 00:08:59:29, 00:09:00:02, 00:09:00:03, ..., 00:09:59:29, 00:10:00:00, 00:10:00:01, etc... Contando os quadros perdidos implica que 00:10:00:00 em perda de quadro iguala 00:10:00:00 em tempo real.

Exemplos 

ShowSMPTE(offset="00:00:59:29", x=360, y=576, font="georgia", size=24, text_color=$ff0000)
# puxará o número do quadro no lado esquerdo do clipe com um compensador de 9 quadros, rolando de cima para baixo
Mpeg2Source("clip.d2v") # sempre é primeiro campo de topo
ShowFrameNumber(scroll=true, offset=9, text_color=$ff0000)
