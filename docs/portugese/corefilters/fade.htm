FadeOut faz o fluxo do vídeo desvanecer linearmente ao preto ou à cor RGB especificada no fim. Semelhantemente FadeIn causa o desvanecer linear a partir do preto ou da cor RGB especificada no início. FadeIO é uma combinação das respectivas funções de FadeIn e FadeOut. A trilha sonora (se presente) também desvanece linearmente para o silêncio ou dele. O desvanecimento afeta só os últimos num_frames  do vídeo. O último quadro do vídeo fica quase preto (ou a cor especificada). Um quadro adicional perfeitamente preto (ou a cor especificada) é incluído no fim, aumentando a conta total dos quadros em um.

FadeIn0 / FadeOut0 não inclui o quadro extra. É útil para processar clipes só de áudio ou para encadear dois ou mais desvanecimentos para obter uma lei quadrada ou cúbica desses efeitos . Por exemplo, Clip.FadeOut0(60) .FadeOut0(60) .FadeOut(60) dá um ataque muito mais afiado e termina mais suave. O ponto de 50% está no quadro 12 do desvanecer, no quadro 30 o desvanecer é 12.5%, no quadro 45, 1.6% - a efetividade é mais pronunciada com áudio.

FadeIn2 / FadeOut2 trabalha igual, exceto  que dois quadros pretos (ou outra cor) são incluídos no fim em vez de um. O propósito principal disto é para contornar um bug do Windows Media Player. Todas as versões de WMP que testei não reproduzem o último quadro de um arquivo MPEG - param próximo do último quadro no fim do playback. Isto deixa um feio quadro, não bastante preto na tela quando o filme termina, se você usar FadeOut. FadeOut2 evita este problema.

O parâmetro color é opcional, default=0 <black> e é especificado como um valor RGB embora o formato de clipe seja baseado em RGB ou YUV.

O parâmetro fps é opcional, default=24.0 e provê uma referência para num_frames em clipes só de áudio. É ignorado se um vídeo estiver presente. Fixa Fps=AudioRate() se o posicionamento da amostra exata de áudio é requerido.

FadeOut(clip, n) é apenas uma abreviação para Dissolve(clip, Blackness (clip, n+1, color=$000000), n) (ou em vez de n+1, n+2 para FadeOut2 e n para FadeOut0).
