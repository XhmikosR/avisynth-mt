Sintaxe

1) SSHiQ (clip, int rY, int rC, int tY, int tC, int str, bool HQY, bool HQC, bool field)

  • rY e rC são respectivamente os raios para luminância e crominância (quanto distante do próximo píxel atual é usado). Maior, mais suave.
  • tY e tC são percentuais (sensibilidade de detalhes). Quanto maior, mais liso (não exatamente igual a outro fpr filtro SSHiQ).
  • str é a força (usado só em modo HiQ), o quanto da versão suavizada deve ser usado para melhor cálculo do novo píxel
  • HQY e HQC contam como usar o modo HiQ, onde as informações de extremidade reduzem o valor de força automaticamente.
    (HQ == 0) ? smoothed_pixel : ( (strength-edge)*smoothed_pixel + (256-(strength-edge))*pixel_in )/256
  • field permite processar campos independentementes (riscos de pior sangria de borrão / cor)

2) SmoothUV (usado para reduzir arco-íris, como faz smartsmoothIQ)
SmoothUV (clip, int radius, int threshold, bool field)

  • radius é a gama de operação
  • threshold é o usado em smartsmoothIQ
  • field é igual ao acima

Este filtro não processa nada do luma e pode ser visto como uma versão mais curta de SSHiQ(rY, radius, -1, threshold, 255, false, false, field)
