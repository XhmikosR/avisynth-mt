1) SSHiQ (clip, int rY, int rC, int tY, int tC, int str, bool HQY, bool HQC, bool field)

  • rY a nebo rC jsou radiusy pro jas - luminance a barevnost - chrominance (jak daleko od aktuálního pixelu jsou použity sousední). Èím vìtší tím vìtší vyhlazení.

  • str je síla (použito jen v HiQ režimu), t.j. kolik vyhlazené verze by mìlo být maximálnì použito pro výpoèet nového pixelu
  • HQY a HQC øíká jestli použít HiQ (vysoká kvalita) režim, kde informace o hranách automaticky snižuje hodnotu síly.
    (HQ == 0) ? smoothed_pixel : ( (strength-edge)*smoothed_pixel + (256-(strength-edge))*pixel_in )/256
  • field umožní vám zpracovat pole nezávisle (riskujete horší smíchání rozmazání/barva)

2) SmoothUV (použito k redukci duhy, jako ve smartsmoothIQ)

  • radius je rozsah operace
  • threshold je ten samý jako ve smartsmoothIQ
